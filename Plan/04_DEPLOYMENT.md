# 04 — Deployment Spec

## Goal

Containerize the FastAPI app with Docker and deploy it to Fly.io with HTTPS, persistent SQLite, and reproducible steps.

---

## File: `Dockerfile`

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# System deps (none required beyond base — SQLite is bundled with Python)
RUN apt-get update && apt-get install -y --no-install-recommends \
    curl ca-certificates \
 && rm -rf /var/lib/apt/lists/*

# Python deps first (cache-friendly)
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# App code
COPY app /app/app

# Create data dir (will be mounted by Fly volume in prod)
RUN mkdir -p /data
ENV DATABASE_URL=sqlite:////data/app.db

EXPOSE 8000

HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
  CMD curl -fsS http://localhost:8000/health || exit 1

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

## File: `fly.toml`

Generated by `fly launch` then edited. Final version:

```toml
app = "happyrobot-carrier-sales"
primary_region = "iad"

[build]

[env]
  ENVIRONMENT = "production"
  LOG_LEVEL = "INFO"
  DATABASE_URL = "sqlite:////data/app.db"

[[mounts]]
  source = "carrier_data"
  destination = "/data"

[http_service]
  internal_port = 8000
  force_https = true
  auto_stop_machines = "stop"
  auto_start_machines = true
  min_machines_running = 0
  processes = ["app"]

  [http_service.concurrency]
    type = "requests"
    soft_limit = 200
    hard_limit = 250

[[http_service.checks]]
  grace_period = "10s"
  interval = "30s"
  method = "GET"
  timeout = "5s"
  path = "/health"

[[vm]]
  size = "shared-cpu-1x"
  memory = "256mb"
```

**Notes**:
- Replace `happyrobot-carrier-sales` with whatever name `fly launch` registered (must be globally unique on Fly).
- `force_https = true` redirects HTTP → HTTPS automatically. Combined with Fly's free LetsEncrypt cert this satisfies the HTTPS requirement.
- `auto_stop_machines = "stop"` saves money — the machine sleeps when idle and cold-starts on first request (~1-2s). Disable for demo if you want zero latency on first call.

## Deployment Steps (First Time)

### Prerequisites
1. Install `flyctl`: `curl -L https://fly.io/install.sh | sh`
2. Sign up + log in: `fly auth login`
3. Add a payment method on Fly (free tier covers this demo, but card is required)

### Step-by-step

```bash
# 1. Initialize Fly app (creates fly.toml — overwrite with the spec version above)
fly launch --no-deploy --name happyrobot-carrier-sales --region iad

# 2. Create persistent volume for SQLite
fly volumes create carrier_data --size 1 --region iad

# 3. Generate an API key and set as secret
export API_KEY=$(python -c "import secrets; print(secrets.token_hex(16))")
echo "Generated API key: $API_KEY  ← save this!"
fly secrets set API_KEY=$API_KEY

# 4. Set FMCSA webkey (once you have it from FMCSA portal)
fly secrets set FMCSA_WEBKEY=<your-fmcsa-webkey>

# 5. Optionally enable mock mode while waiting for FMCSA key
fly secrets set ALLOW_MOCK_FMCSA=true

# 6. Deploy
fly deploy

# 7. Verify
curl https://happyrobot-carrier-sales.fly.dev/health
# Expect: {"status":"ok"}

curl -H "X-API-Key: $API_KEY" https://happyrobot-carrier-sales.fly.dev/loads/search
# Expect: list of loads from CSV

# 8. Open the dashboard
open https://happyrobot-carrier-sales.fly.dev/
# Enter the API key when prompted
```

## Reproduction (from scratch — for the README)

The README must include this section. Anyone with the repo and Fly account should be able to redeploy in <10 minutes:

```bash
# Clone
git clone https://github.com/<user>/happyrobot-carrier-sales.git
cd happyrobot-carrier-sales

# Install flyctl, log in
curl -L https://fly.io/install.sh | sh
fly auth login

# Launch (use your own unique name)
fly launch --no-deploy --name <your-app-name> --region iad
# When prompted, choose: no Postgres, no Redis, copy existing fly.toml=Yes

# Volume
fly volumes create carrier_data --size 1 --region iad

# Secrets
fly secrets set API_KEY=$(python -c "import secrets; print(secrets.token_hex(16))")
fly secrets set FMCSA_WEBKEY=<your-key>

# Deploy
fly deploy

# Check
fly status
fly logs
```

## Operational Commands

```bash
fly logs                       # tail logs
fly ssh console                # shell into the machine
fly secrets list               # show secret names (not values)
fly status                     # health + machine state
fly scale memory 512           # bump RAM if needed
fly restart                    # force a restart
fly destroy <app-name>         # nuke everything (careful)
```

To peek at the SQLite DB on a live machine:
```bash
fly ssh console
sqlite3 /data/app.db
.tables
SELECT * FROM call ORDER BY created_at DESC LIMIT 5;
```

(SQLite CLI is not in the slim image by default — install with `apt-get install -y sqlite3` inside the ssh session if needed.)

## Security Notes (for the docs/Acme deliverable)

What's implemented:
- HTTPS via Fly's automatic Let's Encrypt cert
- HTTP → HTTPS redirect (`force_https`)
- API key auth on every data endpoint
- Secrets managed via `fly secrets` (encrypted at rest, injected as env vars)
- CORS limited at the application layer (currently `*` for demo — narrow to known origins in prod)

What would be added in prod:
- Per-tenant API keys with rotation
- Rate limiting (Fly has a built-in option, or use `slowapi`)
- WAF / DDoS protection (Fly has basic, Cloudflare in front for more)
- Audit logging of all API access
- Separate read-only key for dashboard, write key for HappyRobot
- Network policy: only allow HappyRobot's egress IPs to hit `/calls` POST

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| `fly deploy` succeeds but `/health` 502s | App crashed on startup | `fly logs` — look for missing env vars or CSV path |
| Calls don't persist across redeploys | Volume not mounted | Confirm `[[mounts]]` section in fly.toml |
| FMCSA returns 401/403 | Webkey wrong or not approved yet | Check `fly secrets list`; confirm key in FMCSA portal |
| Dashboard shows "auth failed" | Wrong API key entered, or `API_KEY` secret not set | Check secret, re-enter in dashboard prompt |
| Cold start latency on first call | `auto_stop_machines = stop` | Set to `false` or set `min_machines_running = 1` during demo window |
| HappyRobot HTTP action times out | App on cold start | Set `min_machines_running = 1` (~$2/mo) during the demo |
